# 后续迭代路线图

> V2.0 的可能方向、技术债务、扩展性考虑

---

## 背景

v1.5 功能冻结后，Project Judge 进入稳定期。本文规划了 V2.0 的可能方向，同时记录了当前版本的技术债务。

## V2.0 功能规划

### P0：必须做

#### 1. 云端存储
**现状**：历史记录存在本地（wx.setStorageSync），最多 30 条，换手机就丢失。
**方案**：迁移到微信云数据库，支持跨设备同步。
**技术要点**：
- 云数据库集合设计（evaluations、users）
- 数据迁移脚本（从本地存储导入到云端）
- 离线缓存策略（无网络时仍可查看历史）

#### 2. 用户系统
**现状**：无用户概念，所有数据共享本地存储。
**方案**：微信登录 + 云数据库用户表。
**技术要点**：
- wx.login 获取 openid
- 云数据库按 openid 隔离数据
- 用户设置（API Key 自定义、偏好配置）

#### 3. 真实使用数据统计
**现状**：STAR 简历中的量化数据（如"累计评估 100+ 项目"）是 AI 编造的。
**方案**：接入真实统计数据，注入 Prompt。
**技术要点**：
- 云数据库记录每次评估的 timestamp、category、score
- 统计真实评估次数、平均响应时间、解析成功率
- 将统计数据注入 Prompt 的 resume_star 部分

### P1：应该做

#### 4. 搜索验证 similar_projects
**现状**：similar_projects 依赖 AI 生成，可能编造不存在的项目。
**方案**：接入搜索 API（如 GitHub Search）验证项目是否存在。
**技术要点**：
- AI 输出 similar_projects 后，用 GitHub Search API 验证
- 如果项目不存在，标记为"AI 推测"而非"真实竞品"
- 增加"搜索更多同类项目"功能

#### 5. Benchmark 数据动态更新
**现状**：benchmark.json 是静态文件，数据可能过时。
**方案**：定期从真实数据源更新基准值。
**技术要点**：
- 接入 GitHub Star 分布数据（按类别统计）
- 接入 Product Hunt / Hacker News 评分数据
- 定时任务（每周/每月）更新 benchmark.json

#### 6. 多语言支持
**现状**：Prompt 和输出都是中文。
**方案**：增加英文支持。
**技术要点**：
- Prompt 模板中文化参数
- 输出语言配置
- UI 国际化（i18n）

### P2：可以做

#### 7. AI 对话追问
**现状**：评估结果是一次性的，无法追问。
**方案**：在结果页面增加对话功能。
**技术要点**：
- 对话上下文管理（将评估结果作为上下文）
- DeepSeek 多轮对话 API
- 对话历史存储

#### 8. 项目库管理
**现状**：每次评估都是独立的，无法管理已评估的项目。
**方案**：项目库功能，支持标签、分组、对比。
**技术要点**：
- 云数据库项目表
- 标签系统
- 多维度对比视图

#### 9. 团队协作
**现状**：单人使用。
**方案**：支持团队共享评估结果。
**技术要点**：
- 团队表 + 成员管理
- 共享链接生成
- 权限控制（查看/编辑）

## 扩展性考虑

### 架构扩展

**当前瓶颈**：云函数单实例并发 = 1，DeepSeek API rate limit。

**扩展路径**：
1. **短期**（100 用户）：当前架构够用
2. **中期**（1000 用户）：申请 DeepSeek 付费版提升 rate limit，引入请求队列
3. **长期**（10000+ 用户）：迁移到独立后端（Node.js/Go），支持水平扩展

### 平台扩展

**当前限制**：微信小程序生态绑定。

**扩展路径**：
1. **H5 版本**：Taro 编译到 H5，分享链接直接打开
2. **React Native**：Taro 编译到 RN，支持 iOS/Android
3. **Web 版本**：独立 React 应用，支持更丰富的交互

## 面试官可能追问

1. "V2.0 最优先做什么？"
2. "技术债务怎么排序？"
3. "如果用户量增长 100 倍，架构怎么改？"

## 标准回答

> **Q：V2.0 最优先做什么？**
>
> 云端存储 + 用户系统。原因：当前本地存储限制了产品的可用性（换手机丢失数据、最多 30 条记录）。一旦有了云端存储，其他功能（项目库、团队协作、数据统计）都有了基础。这两个功能的技术复杂度不高（微信云数据库 + wx.login），但对用户体验的提升最大。
