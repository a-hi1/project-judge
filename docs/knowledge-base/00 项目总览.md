## 一句话定位

**AI 项目决策助手** —— 3 分钟判断一个项目是否值得做。

从最初的"AI 评分工具"演化为"AI 项目决策助手"，这个定位变化贯穿了整个产品的设计哲学。详见 [[01 产品设计演化]]。

## 技术栈

| 层级 | 技术选型 | 版本 | 选型理由 |
|------|---------|------|---------|
| 前端框架 | Taro + React | 4.2.0 / 18.3.1 | 跨端能力，微信小程序生态成熟 |
| 开发语言 | TypeScript | 6.0.3 | 类型安全，AI 返回值强校验 |
| 样式方案 | TailwindCSS | 3.4.19 | 快速原型 + 设计系统一致性 |
| 构建工具 | Webpack 5 | via Taro | Taro 默认方案，生态稳定 |
| 后端 | 微信云函数 | wx-server-sdk ~2.6.3 | 免运维，天然鉴权 |
| AI 引擎 | DeepSeek API | deepseek-chat | 性价比极高，中文能力强 |
| 数据源 | GitHub REST API v3 | 未认证 | 5 路并行获取仓库全貌 |
| 存储 | wx.setStorageSync | 本地 | 最多 30 条历史记录 |

详细架构设计见 [[02 架构设计]]。

## 项目文件结构

```
project-judge/
├── cloud/evaluate/
│   ├── index.js          ← 核心：云函数入口 + DeepSeek + GitHub + Benchmark
│   ├── benchmark.json    ← 15 类基准数据
│   └── package.json
├── src/
│   ├── pages/
│   │   ├── home/         ← 输入页 + 最近评估
│   │   ├── result/       ← 完整评估报告
│   │   ├── history/      ← 历史记录列表
│   │   ├── compare/      ← 批量对比（最多 5 个）
│   │   └── settings/     ← API Key 配置说明
│   ├── services/
│   │   ├── ai.ts         ← 云函数调用封装
│   │   └── parser.ts     ← 11+ 容错解析函数
│   ├── types/index.ts    ← TypeScript 接口定义
│   ├── utils/
│   │   ├── theme.ts      ← 设计系统（Linear/Raycast 风格）
│   │   ├── poster.ts     ← Canvas 海报生成
│   │   ├── effort.ts     ← 开发周期规则引擎（已弃用）
│   │   ├── input.ts      ← 输入类型自动检测
│   │   └── score.ts      ← 分数-颜色映射
│   ├── hooks/
│   │   └── useEvaluate.ts ← 评估状态机
│   ├── mock/             ← 契约测试数据
│   └── storage/index.ts  ← 本地存储 CRUD
```

## 关键设计决策速查

| 决策 | 结果 | 原因 |
|------|------|------|
| 前端直调 DeepSeek → 云函数 | ✅ 正确 | API Key 安全 + 免鉴权 |
| 删除 confidence 字段 | ✅ 正确 | AI 打分不稳定，失去用户信任 |
| 增加 Benchmark Engine | ✅ 正确 | 让分数有参考系 |
| effort.ts 规则引擎 | ⚠️ 已弃用 | AI 直接预测更灵活，规则引擎维护成本高 |
| temperature: 0.3 | ✅ 正确 | 平衡创造性与稳定性 |
| response_format: json_object | ✅ 正确 | 强制 JSON 输出，减少解析失败 |
| 本地存储上限 30 条 | ✅ 正确 | 小程序存储限制 + 避免性能问题 |
