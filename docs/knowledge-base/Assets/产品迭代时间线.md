## Week 1: v0.1 — MVP 原型

### 目标
验证核心概念：AI 能否给出有价值的项目评估？

### 实现
- 前端直接调用 DeepSeek API
- Prompt V1：单分评估（score + confidence）
- 一个输入框 + 一个结果页面
- 无历史记录、无设计系统

### 关键决策
- ✅ 选择 DeepSeek（成本低、中文好）
- ❌ 前端直调 DeepSeek（API Key 暴露风险）
- ❌ confidence 字段（不稳定）

### 产出
可工作的 demo，能评估一句话想法

---

## Week 2: v0.3 — 架构迁移

### 目标
解决 API Key 安全问题，增加评估维度

### 实现
- 切换到微信云函数架构
- Prompt V2：六维评分
- 删除 confidence 字段
- 增加 TypeScript 类型定义
- 增加 Parser 容错机制

### 关键决策
- ✅ 云函数中转（API Key 安全）
- ✅ 六维评分（覆盖不同用户画像）
- ✅ 删除 confidence（不可靠）
- ✅ 双层类型设计（AIResponse vs EvaluationResult）

### 产出
安全的架构 + 多维度评估

---

## Week 3: v0.5 — GitHub 集成

### 目标
支持 GitHub 仓库的深度评估

### 实现
- 集成 GitHub REST API（5 路并行）
- 技术栈提取（40+ 已知框架）
- Prompt V3：增加 Benchmark + 同类项目 + category
- 活跃度/社区/影响力/风险四维指标

### 关键决策
- ✅ 5 路并行 API（减少等待时间）
- ✅ Benchmark Engine（分数有参考系）
- ✅ AI 分类 + 规则计算（混合模式）
- ✅ 贡献者数从 Link header 获取（一次请求）

### 产出
GitHub 项目的完整评估报告

---

## Week 4: v0.8 — 功能扩展

### 目标
覆盖求职场景（面试准备 + 简历写法）

### 实现
- Prompt V4：增加面试题 + STAR 简历
- 面试题 5 题 × 6 字段（question/answer/follow_up/difficulty/keywords/category）
- STAR 简历 5 字段（situation/task/action/result/resumeText）
- 历史记录功能（本地存储，最多 30 条）

### 关键决策
- ✅ 面试题的示例格式（提高 answer 质量）
- ✅ STAR 的负面约束（禁止空话）
- ✅ 本地存储 30 条上限（小程序限制）

### 产出
求职场景完整覆盖

---

## Week 5: v1.0 — 决策闭环

### 目标
从"评估工具"升级为"决策助手"

### 实现
- Prompt V5：增加 estimated_dev_time + development_breakdown
- 批量对比功能（最多 5 个项目）
- 分享海报功能（Canvas 2D, 1080px）
- 输入类型自动检测（GitHub/description/idea）

### 关键决策
- ✅ 批量对比（覆盖选择场景）
- ✅ Canvas 海报（分享传播）
- ✅ verdict 规则化（score >= 60）
- ❌ effort.ts 规则引擎（被 AI 直接预测替代）

### 产出
决策链路闭环：评估 → 评分 → 参考 → 行动建议 → 面试准备 → 分享

---

## Week 6: v1.5 — 功能冻结

### 目标
优化输出质量，冻结功能

### 实现
- Prompt V6：优化约束 + 示例格式 + 字数限制
- 契约测试（4 个测试场景）
- 设计系统完善（Linear/Raycast 风格）
- Bug 修复 + 性能优化

### 关键决策
- ✅ 功能冻结（维护成本 > 新功能价值）
- ✅ Prompt 约束收紧（字数限制 + 负面约束）
- ✅ Benchmark dead code 发现但不修复（优先级低）

### 产出
稳定版 v1.5，功能完整，质量可控
