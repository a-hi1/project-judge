# 数据流图

> Project Judge 的完整数据流，从用户输入到页面渲染

---

## 场景一：GitHub 项目评估

```
用户输入: "https://github.com/owner/repo"
  │
  ▼
┌─────────────────────────────┐
│ input.ts: detectInputType() │  自动检测输入类型
│ 匹配 github.com URL         │
└──────────┬──────────────────┘
           │ type = "github"
           ▼
┌─────────────────────────────┐
│ ai.ts: evaluate()           │  调用云函数
│ wx.cloud.callFunction({    │
│   name: 'evaluate',        │
│   data: { content, type }  │
│ })                          │
└──────────┬──────────────────┘
           │
           ▼
┌═══════════════════════════════════════════════════════════┐
║                  云函数 evaluate/main()                    ║
║                                                           ║
║  ┌─────────────────────────────────────────────────────┐  ║
║  │ fetchGitHubRepo(url)                                │  ║
║  │                                                     │  ║
║  │  Promise.all([                                      │  ║
║  │    GET /repos/:owner/:repo          → 元数据        │  ║
║  │    GET /repos/:owner/:repo/readme   → README(3000字) │  ║
║  │    GET .../contents/package.json    → 技术栈        │  ║
║  │    GET .../contributors?per_page=1  → 贡献者数      │  ║
║  │    GET .../releases?per_page=1      → 发布数        │  ║
║  │  ])                                                 │  ║
║  │                                                     │  ║
║  │  → repoInfo = {                                     │  ║
║  │      name, description, stars, forks, language,     │  ║
║  │      topics, updated_at, open_issues, readme,       │  ║
║  │      techStackFromPkg,                              │  ║
║  │      githubAnalysis: {                              │  ║
║  │        stars, forks, openIssues, contributors,      │  ║
║  │        lastCommit, releases, license,               │  ║
║  │        activityLevel, communityLevel,               │  ║
║  │        influenceLevel, riskLevel                    │  ║
║  │      }                                              │  ║
║  │    }                                                │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  ┌──────────────────────▼──────────────────────────────┐  ║
║  │ getSystemPrompt('github', repoInfo)                 │  ║
║  │                                                     │  ║
║  │ 注入:                                               │  ║
║  │  仓库名 + 描述 + Stars + Forks + 语言              │  ║
║  │  + Topics + 最近更新 + Open Issues                  │  ║
║  │  + 技术栈 + README 摘要                             │  ║
║  │                                                     │  ║
║  │ 约束: "基于真实数据评估，不要编造"                    │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  ┌──────────────────────▼──────────────────────────────┐  ║
║  │ callDeepSeek(content, 'github', repoInfo)           │  ║
║  │                                                     │  ║
║  │ POST api.deepseek.com/chat/completions              │  ║
║  │ {                                                   │  ║
║  │   model: "deepseek-chat",                           │  ║
║  │   temperature: 0.3,                                 │  ║
║  │   response_format: { type: "json_object" },         │  ║
║  │   messages: [                                       │  ║
║  │     { role: "system", content: systemPrompt },      │  ║
║  │     { role: "user", content: userInput }            │  ║
║  │   ]                                                 │  ║
║  │ }                                                   │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  ┌──────────────────────▼──────────────────────────────┐  ║
║  │ extractJson(messageContent)                         │  ║
║  │                                                     │  ║
║  │ 1. 直接 JSON.parse(cleaned)                         │  ║
║  │ 2. 去掉 ```json ... ``` 后 parse                    │  ║
║  │ 3. 提取 { 到 } 之间的内容后 parse                   │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  ┌──────────────────────▼──────────────────────────────┐  ║
║  │ calculateBenchmark(recScore, category)              │  ║
║  │                                                     │  ║
║  │ category = result.category || 'default'             │  ║
║  │ data = benchmarkData[category]                      │  ║
║  │                                                     │  ║
║  │ if score >= top10 → "Top 10%"                       │  ║
║  │ if score >= top20 → "Top 20%"                       │  ║
║  │ if score >= top30 → "Top 30%"                       │  ║
║  │ if score >= avg   → "Top 50%"                       │  ║
║  │ else              → "Bottom 50%"                    │  ║
║  │                                                     │  ║
║  │ → 覆盖 result.benchmark                             │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  ┌──────────────────────▼──────────────────────────────┐  ║
║  │ result.github_analysis = repoInfo.githubAnalysis    │  ║
║  └──────────────────────┬──────────────────────────────┘  ║
║                         │                                  ║
║  return { success: true, data: result }                   ║
╚═════════════════════════╤═════════════════════════════════╝
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│ parser.ts: transformResponse(raw)                       │
│                                                         │
│  raw (AIResponse, snake_case)                           │
│    │                                                    │
│    ├─→ project_name ──→ projectName (ensureString)      │
│    ├─→ verdict ──→ normalizeVerdict()                   │
│    ├─→ estimated_dev_time ──→ normalizeEstimatedDevTime │
│    ├─→ scores.resume.score ──→ clampScore()             │
│    ├─→ scores.learning.score ──→ clampScore()           │
│    ├─→ scores.business.score ──→ clampScore()           │
│    ├─→ scores.difficulty.score ──→ clampScore()         │
│    ├─→ scores.innovation.score ──→ clampScore()         │
│    ├─→ scores.recommendation.score ──→ clampScore()     │
│    ├─→ similar_projects ──→ parseSimilarProjects()      │
│    ├─→ benchmark ──→ parseBenchmark()                   │
│    ├─→ roadmap ──→ parseRoadmap()                       │
│    ├─→ interview_questions ──→ parseInterviewQuestions() │
│    ├─→ resume_star ──→ parseResumeStar()                │
│    ├─→ development_breakdown ──→ parseDevelopmentBreak..│
│    ├─→ github_analysis ──→ parseGitHubAnalysis()        │
│    └─→ fillDefaultValues() ──→ 补全所有缺失字段         │
│    │                                                    │
│    ▼                                                    │
│  EvaluationResult (camelCase, 严格类型)                 │
└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│ 页面渲染                                                 │
│                                                         │
│  Result Page:                                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │ Hero Card: projectName + score + verdict        │   │
│  │ ScoreBar × 6: 五维评分可视化                     │   │
│  │ Benchmark: percentile + average                  │   │
│  │ Similar Projects: name + similarity + why       │   │
│  │ GitHub Analysis: activity/community/influence   │   │
│  │ Interview Questions: Q + A + follow-up × 5      │   │
│  │ Resume Star: S + T + A + R + resumeText         │   │
│  │ Development Breakdown: task + days              │   │
│  │ Roadmap: day + task                             │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 场景二：一句话想法评估

```
用户输入: "做一个 AI 自动生成周报的工具"
  │
  ▼
detectInputType() → type = "idea" (短文本，非 URL)
  │
  ▼
云函数: fetchGitHubRepo() 不调用
  │
  ▼
getSystemPrompt('idea', null)
  intro = "用户给出了一个项目想法，评估可行性、市场潜力、技术难度"
  │
  ▼
其余流程与 GitHub 场景相同
  │
  ▼
github_analysis = null (非 GitHub 项目)
readme_summary = "" (非 GitHub 项目)
```

## 场景三：批量对比

```
用户输入: 5 个项目（每行一个）
  │
  ▼
逐个调用云函数（顺序执行，非并行）
  │
  ├─→ evaluate("项目1") → result1
  ├─→ evaluate("项目2") → result2
  ├─→ evaluate("项目3") → result3
  ├─→ evaluate("项目4") → result4
  └─→ evaluate("项目5") → result5
  │
  ▼
按 recommendation.score 降序排序
  │
  ▼
Compare Page:
  ┌─────────────────────────────────┐
  │ 🥇 项目3 (score: 85) ← 高亮   │
  │ 🥈 项目1 (score: 78)           │
  │ 🥉 项目5 (score: 72)           │
  │    项目2 (score: 65)            │
  │    项目4 (score: 58)            │
  └─────────────────────────────────┘
```
