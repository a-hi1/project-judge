# 海报系统设计

> Canvas 2D 生成 1080px 可分享海报

---

## 背景

评估结果出来后，用户可能想分享到朋友圈或微信群。Project Judge 提供了"生成海报"功能，将评估结果渲染为一张 1080px 宽的图片，保存到相册。

## 设计目标

1. 生成一张视觉精美的评估海报
2. 包含核心信息：项目名、总分、结论、Benchmark、五维评分、同类项目、成功因素、MVP 时间线
3. 1080px 宽度，适合朋友圈分享
4. 一键保存到相册

## 实现细节

### 技术方案

**文件**：`src/utils/poster.ts`（约 280 行）

使用 **Canvas 2D API**（非旧版 CanvasContext）渲染海报。

**为什么用 Canvas 2D？**
- Taro 的 `Canvas 2D` 支持更丰富的绘制操作
- 性能更好（硬件加速）
- 支持 `canvasToTempFilePath` 导出为图片

### 关键实现

**1. Canvas 初始化**

```typescript
const query = Taro.createSelectorQuery()
query.select('#posterCanvas')
  .fields({ node: true, size: true })
  .exec((res) => {
    const canvas = res[0].node
    const ctx = canvas.getContext('2d')
    // 设置 canvas 尺寸
    canvas.width = 1080
    canvas.height = 1920  // 16:9 比例
  })
```

**2. 绘制流程**

按层级从上到下绘制：
1. 背景色（#0A0A0A）
2. 项目名称（大字）
3. 分数（超大字号 88rpx）
4. 结论 + Benchmark
5. 五维评分条（进度条样式）
6. 同类项目列表
7. 成功因素列表
8. MVP 时间线
9. 底部签名

**3. 导出图片**

```typescript
Taro.canvasToTempFilePath({
  canvas: canvas,
  quality: 1,
  fileType: 'png',
  width: 1080,
  height: actualHeight,
  success: (res) => {
    Taro.saveImageToPhotosAlbum({
      filePath: res.tempFilePath,
      success: () => Taro.showToast({ title: '已保存到相册' }),
    })
  },
})
```

### 设计决策

**1. 为什么 1080px？**
- 朋友圈图片推荐宽度：1080px
- 不会模糊（Retina 屏幕 2x = 540pt）
- 不会太大（文件大小约 200-500KB）

**2. 为什么暗色主题？**
- 与 App 内的视觉风格一致
- 暗色背景下的白色文字对比度高
- 在朋友圈中更有辨识度

**3. 为什么不用 html2canvas？**
- 微信小程序环境不支持 DOM 操作
- Canvas 2D 是小程序唯一可靠的图片生成方案
- 性能更好，不依赖第三方库

## 优缺点分析

| 优点 | 缺点 |
|------|------|
| 1080px 高清，朋友圈不模糊 | Canvas 绘制代码冗长（280 行） |
| 暗色主题辨识度高 | 布局调整需要修改代码，不灵活 |
| 一键保存到相册 | 不同内容长度需要动态计算画布高度 |
| 不依赖第三方库 | 图片中不支持超链接 |

## 面试官可能追问

1. "Canvas 绘制的性能怎么样？"
2. "海报内容多了画布装不下怎么办？"
3. "为什么不做成模板引擎？"

## 标准回答

> **Q：海报内容多了画布装不下怎么办？**
>
> 当前方案是动态计算画布高度。每个模块绘制完后，记录当前 Y 坐标，最后用实际 Y 坐标作为画布高度。如果内容特别多（比如 5 个同类项目 + 长 MVP 时间线），画布可能超过 3000px。这种情况下会截断底部内容，并在底部加"查看更多请打开小程序"的提示。未来可以考虑分页生成（多张图片）或让用户选择分享哪些模块。
